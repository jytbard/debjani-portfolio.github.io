Requirements

The projects' purpose is to create a customer service chatbot for our website. The chatbot will interface with several Data sets:

 

one with participant information stored in SQL

 

one with plan data (consisting of plan summaries in PDF Format, excel lists, etc)

 

possible third set that retrieves data from Microsoft dynamics

 

the participant data and transactions, currently stored in on-premise servers, are copied and pushed to the cloud (azure or AWS) each night. THis cloud stored data is then accessible to the website and employer portal websites. At Present, there's no mechanism for data to be sent back from the cloud to the on-premise servers, but potential changes are being explored to allow for features like distribution requests and employee status updates.

 

the chatbots workflow is as follows:

 

1- The chatbot prompts the participant to input their Social Security Number (SSN) upon logging in. It then retrieves the participants plan information based on the providers SSN from the SQL data. TH chatbot asks the participant which plan they want to discuss and can provide more plan details if required by retrieving additional information from plan data or Microsoft Dynamics.

 

2- The Participant then asks their questions. The nature of these questions can range from inquiring about their account balance to asking about distribution options and tax related matters. If the question concerns their account balance, the chatbot calls the SQL data to confirm if the participant is eligible for distribution.

 

3- depending on the circumstances, the bot might email the employer contact to check if the participant has been terminated or generate an employer portal record asking the employer to update the status.

 

4- The chatbot retrieves the appropriate forms for the plan/ plan type and makes them available for download, or send via email. It may also potentially allow the participant to enter details and generate distribution form.

 

5- For tax related questions, the chatbot retrieves forms or consults GPT-4 API for knowledge on tax laws

 

6- for inquiries about plan details, the bot retrieves data from the plan summary.

 

the goal of this integration is to provide more seamless and interactive user experience for participants, assiting them with relevant information and helping them manage their plans more effectively
